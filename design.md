# Design Guidelines

## 参考サイト

https://thebeauteclinic.jp/

上記サイトのデザインテイスト・レイアウト・雰囲気・UXを全面的に参考にすること。
実装前に必ず参考サイトを確認し、そのトーン・マナーに沿ったUIを作成すること。
※ カラーはホワイトベースに変更しているため、色に関してはこのガイドラインを優先すること。

---

## デザインの方向性

- **テイスト**: ラグジュアリー / ハイエンド美容 / エレガント
- **トーン**: 洗練された・上品・清潔感と華やかさの共存
- **背景**: ホワイト〜オフホワイトをベースに、ブラックとゴールドで引き締める
- **余白**: 広め。余白が「高級感」を演出するため、詰め込まない
- **英日混在**: セクション見出しは英語大文字メイン＋日本語サブテキストの組み合わせ

## カラーパレット

| 用途                   | カラー                                       |
| ---------------------- | -------------------------------------------- |
| 背景（メイン）         | `#FFFFFF` 〜 `#F8F7F4`（オフホワイト）       |
| 背景（セクション差分） | `#F2F0EC` 〜 `#EDEBE6`                       |
| テキスト（メイン）     | `#1A1A1A`                                    |
| テキスト（サブ）       | `#888888` 〜 `#AAAAAA`                       |
| アクセント（ゴールド） | `#C9A96E` 〜 `#D4B483`（落ち着いたゴールド） |
| ボーダー・区切り線     | `#E0DDD8` 〜 `#D8D4CE`                       |

> 蛍光色・ビビッドカラーは一切使わない。ゴールドも派手にならないよう彩度を抑えること。

## レイアウト

- フルワイドセクション構成（各セクションが画面幅いっぱいに広がる）
- コンテンツ幅は最大 `1200px`、センタリング
- セクション間余白は広め（`padding: 100px〜160px 0`）
- ヒーローは全画面（`100vh`）、動画または大きい画像背景
- グリッドは2カラム中心（左: テキスト / 右: 画像 など）
- モバイルファーストで実装

---

## セクション構成の参考（thebeauteclinic.jp より）

各セクションの見出しパターン：

```
[英語セクション名（小文字）]  【 日本語セクション名 】
例: menu 【 施術メニュー 】
例: doctor 【 医師紹介 】
```

典型的なページ構成：

1. Hero（全画面動画 or 画像 ＋ キャッチコピー）
2. 特集・メインコンテンツ
3. Doctor（医師・担当者紹介）
4. Menu / Service（サービス一覧）
5. Case / Works（実績・事例）
6. SNS（公式アカウント）
7. News
8. ループテキスト帯（マーキー）
9. Campaign / Blog
10. Contact / Footer

## 画像の使い方

### プロジェクト内の画像

- `img/` フォルダ内の画像を最優先で使用すること
- 実装前に必ず `img/` フォルダの中身を確認する
- 全画面ヒーロー・縦長ポートレート・横長バナーなど用途に合わせてトリミング

### フリー素材

- `img/` フォルダに適切な素材がない場合はフリー素材をダウンロードして使用すること
- 推奨サイト（ラグジュアリー・美容系の雰囲気に合うもの）:
  - [Unsplash](https://unsplash.com) — キーワード: `beauty clinic`, `luxury aesthetic`, `skincare`, `elegant`
  - [Pexels](https://pexels.com) — 同上
- ダウンロードした画像は `img/` フォルダに保存してから使用すること

## UIコンポーネント

### ボタン（CTA）

- フィルドボタン（黒背景・白文字）またはゴールドボタン
- アウトラインボタン使用時は黒枠・黒文字
- 角丸なし〜ごく控えめ（`border-radius: 0〜2px`）
- `letter-spacing: 0.15em` で上品に
- ホバー時: ゴールドに変化、`transition: 0.4s ease`

### カード・一覧

- 影なし、背景色の差分かボーダーで区切る
- 画像 ＋ テキスト構成
- ホバー時は画像がわずかにスケールアップ（`transform: scale(1.03)`）

### ナビゲーション

- 白背景（スクロールで白背景 ＋ 薄いボーダーに変化）
- ロゴ左 / メニュー右の配置
- 文字は小さめ・スペーシング広め
- ハンバーガーメニュー（モバイル）: シンプルな3本線

### マーキー（ループテキスト帯）

- 水平スクロールするブランドコピー・キーワードの帯
- 背景をオフホワイト or ゴールドにして視覚的なアクセントに
- 速度は遅め（`20s〜30s` で1周）

---

## アニメーション・インタラクション

- 動きは上品・控えめに
- スクロール時のフェードイン（`opacity: 0 → 1` ＋ `translateY(20px → 0)`）
- ヒーロー動画はautoplay / muted / loop
- マーキーのループスクロールはOK
- その他のループアニメーションは使わない
- `transition` の duration は `0.4s〜0.6s`

## Figma連携（MCP）について

このUIは最終的に **Figma MCP** を使用してFigma上にデザインとして書き出す予定。
以下を徹底すること：

- **CSSカスタムプロパティ**で色・フォント・スペーシングを一元管理する
  ```css
  :root {
    --color-bg: #f8f7f4;
    --color-bg-section: #f2f0ec;
    --color-text: #1a1a1a;
    --color-text-sub: #888888;
    --color-accent: #c9a96e;
    --color-border: #e0ddd8;
    --font-en: "Cormorant Garamond", serif;
    --font-ja: "Noto Sans JP", sans-serif;
    --spacing-section: 120px;
  }
  ```
- コンポーネントは明確に分割し、セクション単位でグループ化する（FigmaのFrame単位を意識）
- **アニメーションはFigmaに書き出せない**ため、「アニメーションOFF状態＝デフォルト表示状態」を必ず定義する
  - フェードインで `opacity: 0` から始まる要素は、Figma書き出し用に `opacity: 1` のスタイルをデフォルトに持たせる
  - hover状態など複数の状態は、別クラスで静的スタイルを定義する（FigmaのVariantとして書き出しやすくなる）

## やってはいけないこと

- 黒・暗い背景をベースにしない（ホワイト基調を守る）
- 蛍光色・ビビッドカラーを使わない
- フォントを3種類以上使わない
- 影（drop-shadow）を多用しない
- アニメーションを多用しない
- 情報を詰め込みすぎない（余白が高級感を作る）
- 角丸を多用しない（シャープな印象を保つ）
